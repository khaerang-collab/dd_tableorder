# dd_tableorder - 테이블 오더 시스템

## 프로젝트 소개

식당에서 QR코드나 태블릿을 통해 고객이 직접 메뉴를 주문할 수 있는 **디지털 테이블 오더 시스템**입니다.
같은 테이블에 앉은 여러 고객이 동시에 장바구니를 공유하며 주문할 수 있고, 관리자는 실시간으로 주문을 모니터링하고 관리할 수 있습니다.

> 참고 디자인: 토스 오더

## 주요 기능

### 고객 기능

#### 1. 테이블 접속
- QR코드 스캔으로 비밀번호 없이 즉시 접속
- 같은 테이블에 여러 명이 동시 접속 가능 (현재 N명 함께 주문 중 표시)
- 태블릿 자동 로그인 지원
- 16시간 세션 유지

#### 2. 메뉴 탐색
- 카테고리별 메뉴 탐색 (상단 고정 탭)
- 리스트형 레이아웃 (텍스트 + 100x100px 썸네일)
- 스크롤 시 카테고리 자동 하이라이트
- 터치 친화적 UI (최소 44x44px 버튼)

#### 3. 공유 장바구니
- 같은 테이블 고객 전원이 하나의 장바구니를 공유
- 내가 담은 항목 / 다른 사람이 담은 항목 구분 표시
- 본인 항목만 수정/삭제 가능
- 실시간 동기화

#### 4. 주문하기
- 주문 전 최종 확인 화면
- 주문 성공 시: 주문번호 표시 -> 5초 카운트다운 -> 메뉴로 자동 이동
- 주문 실패 시: 에러 표시, 장바구니 내용 유지

#### 5. 주문 내역
- 현재 테이블 세션의 전체 주문 내역 조회 (모든 사용자 포함)
- 주문 상태 추적: 대기중 -> 준비중 -> 완료
- 실시간 상태 업데이트

### 관리자 기능

#### 6. 관리자 로그인
- 매장 ID + 아이디 + 비밀번호 인증
- JWT 기반 16시간 세션
- 로그인 시도 횟수 제한 (brute-force 방지)

#### 7. 실시간 주문 모니터링
- SSE 기반 실시간 대시보드 (2초 이내 업데이트)
- 테이블별 카드 레이아웃 (테이블 번호, 총 주문 금액, 최근 주문 미리보기)
- 신규 주문 시 시각적 강조 (색상 변경, 애니메이션)
- 주문 상태 변경 관리

#### 8. 테이블 관리
- 테이블 등록 및 QR코드 생성
- 세션 관리: 시작 (첫 주문 시) -> 종료 (완료 처리)
- 완료 시 주문 내역을 히스토리로 이동 후 초기화
- 과거 주문 이력 조회 (날짜별 필터링)

#### 9. 메뉴 관리
- 메뉴 CRUD (이름, 가격, 설명, 카테고리, 이미지)
- 메뉴 표시 순서 조정
- 카테고리 자유 추가/수정/삭제

#### 10. 이미지 업로드
- 메뉴 등록/수정 시 이미지 파일 업로드
- JPG, PNG 지원
- 이미지 없이도 메뉴 등록 가능

## 데이터 모델 (핵심 엔티티)

| 엔티티 | 설명 |
|--------|------|
| Store | 매장 정보 |
| AdminUser | 관리자 계정 (매장별) |
| Table | 테이블 정보 (매장별) |
| TableSession | 고객 세션 (테이블별) |
| Category | 메뉴 카테고리 (매장별, 동적 관리) |
| Menu | 메뉴 항목 (카테고리별) |
| Order | 주문 (테이블 세션별) |
| OrderItem | 주문 내 개별 항목 |
| OrderHistory | 과거 주문 이력 |
