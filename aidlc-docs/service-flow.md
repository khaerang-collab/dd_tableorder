## 1. 고객(사용자) 서비스 플로우

### 1.1 접속 플로우

```
[테이블 착석]
    │
    ├─── (방식 A) QR 코드 스캔
    │         │
    │     브라우저 URL 접속 (매장ID + 테이블번호 포함)
    │         │
    │     카카오 로그인 화면 ("카카오로 시작" 버튼)
    │         │
    │     ┌───┴───┐
    │  최초 방문   재방문
    │  프로필      기존 프로필
    │  자동 생성   자동 연결
    │  (닉네임,    (카카오 ID
    │   성별,      기반 매칭)
    │   연령대,
    │   프로필
    │   이미지)
    │     └───┬───┘
    │         │
    │     메뉴 화면 진입
    │
    ├─── (방식 B) 태블릿 자동 로그인
    │     (관리자 초기 설정 1회 완료 상태)
    │         │
    │     로그인 정보 로컬 저장 → 메뉴 화면 바로 진입
    │
    └─── 동시 접속 시
          "현재 N명이 함께 주문 중" 안내 실시간 표시
```

**핵심 규칙:**
- 카카오 로그인 필수 (비회원 주문 불가)
- 로그인하지 않으면 메뉴 화면 접근 불가
- 세션 유지 시간: 16시간

### 1.2 메뉴 탐색 플로우

```
[메뉴 화면 - 홈]
    │
    ├── [상단 네비게이션]
    │     ├── 좌측: "직원호출" 버튼 (벨 아이콘)
    │     └── 우측: "주문내역" 텍스트 버튼
    │
    ├── [매장 정보 헤더]
    │     ├── 매장명 / 테이블번호
    │     ├── "멤버도 QR 찍고 함께 주문해요" 안내
    │     └── 매장 공지 메시지
    │
    ├── [맞춤 추천 섹션]
    │     ├── "{닉네임}님 맞춤 추천"
    │     ├── 재방문 시: "지난번에 드신 메뉴"
    │     ├── 첫 방문 시: 매장 인기 메뉴로 대체
    │     └── 유사 매장 이력 기반: "이런 메뉴 어때요?"
    │
    ├── [카테고리 탭] (Sticky 고정, 수평 스크롤)
    │     ├── 탭 클릭 → 해당 섹션으로 스크롤 이동
    │     ├── 스크롤 시 현재 카테고리 탭 자동 하이라이트
    │     └── 드롭다운 → 전체 카테고리 바텀시트
    │
    └── [메뉴 리스트] (카테고리별 섹션)
          ├── 리스트형 레이아웃 (텍스트 좌측 + 썸네일 우측)
          ├── 뱃지: 인기 / 신규
          └── 메뉴 아이템 탭 → 메뉴 상세 화면
```

### 1.3 메뉴 상세 → 장바구니 담기 플로우

```
[메뉴 상세 화면]
    │
    ├── 메뉴명, 가격, 설명, 이미지
    │
    ├── [손님들의 선택] 페어링 추천 (하단 섹션)
    │     ├── 해당 매장 주문 데이터 기반 연관 메뉴 (최대 3개)
    │     ├── 함께 주문된 횟수 표시 (예: "326번 함께 주문됐어요")
    │     ├── 추천 메뉴 클릭 → 해당 메뉴 상세로 이동
    │     └── 데이터 부족 시 섹션 미표시
    │
    └── 수량 선택 → [장바구니 담기]
          │
          ├── 토스트: "장바구니에 메뉴를 추가했어요"
          ├── 하단 플로팅 바 갱신: [🛒 00,000원 장바구니 보기]
          └── 동행자 화면에도 실시간 반영
              토스트: "멤버가 {메뉴명}을 담았어요"
```

### 1.4 장바구니 → 주문 플로우

```
[장바구니 화면] (하단 플로팅 바 탭으로 진입)
    │
    ├── [테이블 공유 장바구니]
    │     ├── 같은 메뉴 → 하나의 행에 병합 표시
    │     ├── 주문자 아바타 아이콘 (카카오 프로필 + 닉네임)
    │     │     └── 본인 항목: "나" 라벨 추가
    │     ├── 각 아이콘에 해당 사용자 수량 표시
    │     ├── 총 수량: 전체 합산
    │     └── 누구나 모든 항목 수량 변경 / 삭제 가능
    │
    ├── [프로모션 넛지] (활성 프로모션이 있을 때만)
    │     ├── 프로그레스 바 + 남은 금액 표시
    │     │     예: "3,000원만 더 주문하면 마카롱 서비스!"
    │     ├── 주문 금액 변동 시 실시간 업데이트
    │     ├── 여러 프로모션 → 가장 가까운 목표부터 표시
    │     └── 목표 달성 시:
    │           ├── 축하 애니메이션 + 문구 (예: "마카롱도 드려요! 🎉")
    │           └── 서비스 메뉴 장바구니 자동 추가 (0원, "서비스" 태그)
    │
    ├── 총 금액 실시간 계산 (전체 접속자 합산)
    │
    └── [주문하기] 버튼
          │
      ┌───┴───┐
   성공        실패
    │           │
    │       에러 메시지 표시
    │       장바구니 유지
    │       재시도 안내
    │
[주문 완료 화면]
    ├── "총 N개 주문 완료!" 메시지
    ├── 주문 번호 표시
    ├── 공유 장바구니 전체 비우기 (모든 접속자 반영)
    ├── 5초 후 메뉴 화면 자동 리다이렉트
    └── "메뉴판으로" 버튼 (수동 이동)

    ※ 추가 주문 시 동일 플로우 반복
```

### 1.5 주문 내역 조회 플로우

```
[메뉴 화면] → 상단 "주문내역" 버튼
    │
[주문 내역 화면]
    ├── 현재 테이블 세션의 전체 주문 표시
    │     (같은 테이블의 모든 접속자 주문 포함)
    ├── 주문별 상세 정보:
    │     ├── 주문 번호
    │     ├── 주문 시각
    │     ├── 메뉴 / 수량
    │     ├── 금액
    │     └── 상태: 대기중 → 준비중 → 완료
    ├── 주문 시간 순 정렬
    ├── 주문 상태 실시간 업데이트 (관리자 변경 시 반영)
    ├── 이용 완료 처리된 이전 세션 주문은 제외
    └── "추가 주문하기" / "메뉴 보기" 버튼 → 메뉴 화면 복귀
```

### 1.6 직원 호출 플로우

```
[메뉴 화면] → 상단 "직원호출" 버튼
    │
[바텀시트] "무엇을 도와드릴까요?"
    │
    ├── 물/물티슈 요청
    ├── 수저/접시 요청
    └── 기타 요청 → 텍스트 입력 가능
          │
      호출 전송 (서버 → SSE → 관리자)
          │
      토스트: "호출했어요"
          │
      30초 쿨다운 (버튼 비활성, 연속 호출 방지)
```

---

### 1.7 세션 종료 플로우

```
[정상 종료]
    관리자가 "이용 완료" 처리
        → 고객 화면에 세션 종료 안내 표시
        → 현재 주문 내역 초기화

[세션 만료]
    16시간 경과
        → 세션 만료 안내 메시지
        → QR 재스캔으로 새 세션 시작

[네트워크 끊김]
    오프라인 상태 안내 메시지 표시
        → 장바구니 로컬 데이터 유지
        → 재접속 시 서버에서 장바구니 복원
```

## 2. 관리자(운영자) 서비스 플로우

### 2.1 매장 오픈 (로그인) 플로우

```
[매장 오픈]
    │
[관리자 로그인 화면]
    ├── 매장 식별자 입력
    ├── 사용자명 입력
    └── 비밀번호 입력
          │
      ┌───┴───┐
   성공        실패
    │           │
    │       에러 메시지 표시
    │       로그인 시도 제한 (brute-force 방어)
    │
JWT 토큰 발급
    │
[주문 대시보드] 자동 이동
    │
    ├── 16시간 세션 유지
    ├── 브라우저 새로고침 시 세션 유지
    └── 16시간 후 자동 로그아웃
```

---

### 2.2 실시간 주문 모니터링 플로우

```
[주문 대시보드]
    │
    ├── [테이블 카드 그리드]
    │     ├── 테이블 번호
    │     ├── 총 주문액
    │     ├── 최신 주문 미리보기
    │     ├── 직원호출 알림 (있을 경우, 사유 포함)
    │     └── 테이블별 필터링 기능
    │
    ├── [SSE 실시간 업데이트] (2초 이내)
    │     ├── 신규 주문 → 해당 카드 시각적 강조 (색상/애니메이션)
    │     ├── 직원 호출 → 카드에 호출 사유 표시 + 아이콘
    │     └── 10분 이상 대기 주문 → 강조 알림 (색상/깜박임)
    │
    └── [테이블 카드 클릭] → 주문 상세 보기
          │
      [테이블 주문 상세]
          ├── 전체 주문 목록
          │     ├── 주문 번호
          │     ├── 주문 시각
          │     ├── 메뉴 / 수량
          │     ├── 금액
          │     └── 상태
          │
          └── [주문 상태 변경] (주문 건 단위)
                │
            대기중 ───→ 준비중 ───→ 완료
            (자동설정)   (수동변경)   (수동변경)
                │
            상태 변경 시 고객 화면에도 실시간 반영 (SSE)
```

### 2.3 테이블 관리 플로우

```
[테이블 관리 화면]
    │
    ├── [테이블 등록 / 초기 설정]
    │     ├── 테이블 번호 입력 및 등록
    │     ├── QR 코드 자동 생성 (매장ID + 테이블번호 URL 인코딩)
    │     └── 16시간 세션 생성
    │
    ├── [주문 삭제] (대시보드 상세에서)
    │     ├── 삭제 버튼 클릭
    │     ├── 확인 팝업 표시
    │     ├── 확인 → 주문 즉시 삭제
    │     ├── 총 주문액 자동 재계산
    │     └── 성공/실패 피드백
    │
    ├── [이용 완료 처리]
    │     ├── "이용 완료" 버튼 클릭
    │     ├── 확인 팝업 표시
    │     ├── 확인 → 처리:
    │     │     ├── 현재 주문 내역 → 과거 이력으로 이동
    │     │     ├── 테이블 현재 주문 목록 리셋
    │     │     ├── 총 주문액 0원으로 초기화
    │     │     └── 고객 화면에 세션 종료 안내
    │     └── 새 고객 → 이전 주문 없이 깨끗한 상태에서 시작
    │
    ├── [직원호출 응대]
    │     ├── 테이블 카드에 호출 알림 표시 (사유 포함)
    │     ├── 여러 테이블 동시 호출 시 각각 독립 표시
    │     └── "확인" 버튼 → 호출 알림 해제
    │
    └── [과거 주문 내역 조회]
          ├── "과거 내역" 버튼으로 접근
          ├── 테이블별 과거 주문 목록 (시간 역순)
          ├── 주문별: 주문번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
          ├── 날짜 필터링 기능
          └── "닫기" → 대시보드 복귀
```

### 2.4 메뉴 / 카테고리 관리 플로우

```
[메뉴 관리 화면]
    │
    ├── [카테고리 관리]
    │     ├── 카테고리 추가: 카테고리명 입력
    │     ├── 카테고리 수정: 카테고리명 변경
    │     ├── 카테고리 삭제: 소속 메뉴 없을 때만 가능
    │     └── 카테고리 노출 순서 조정
    │
    └── [메뉴 관리]
          │
          ├── [메뉴 등록]
          │     ├── 메뉴명 입력 (필수)
          │     ├── 가격 입력 (필수, 0 이상)
          │     ├── 설명 입력
          │     ├── 카테고리 선택 (필수)
          │     ├── 이미지 업로드 (선택, JPG/PNG)
          │     │     ├── 서버에 파일 저장
          │     │     └── 업로드 성공 시 미리보기 표시
          │     └── 필수 필드 및 가격 범위 검증
          │
          ├── [메뉴 수정]
          │     ├── 기존 정보 불러와서 편집
          │     └── 이미지 변경 가능
          │
          ├── [메뉴 삭제]
          │     └── 확인 팝업 후 삭제
          │
          └── [메뉴 노출 순서 조정]
```

### 2.5 프로모션 관리 플로우

```
[프로모션 관리 화면]
    │
    ├── [프로모션 등록]
    │     ├── 목표 금액 설정 (예: 30,000원)
    │     ├── 서비스 메뉴/혜택 이름 입력 (예: "마카롱 1개 서비스")
    │     ├── 활성/비활성 토글 설정
    │     └── 필수 필드 검증 (목표 금액 > 0, 서비스 내용 필수)
    │
    ├── [다단계 프로모션]
    │     여러 프로모션 동시 등록 가능
    │     예) 20,000원 → 음료 서비스
    │         30,000원 → 디저트 서비스
    │
    ├── [프로모션 수정 / 삭제]
    │
    └── [고객 화면 반영 결과]
          ├── 장바구니 플로팅 바에 프로그레스 표시
          ├── 남은 금액 + 프로그레스 바 시각화
          ├── 목표 달성 시 축하 애니메이션 + 서비스 자동 추가
          ├── 금액 미달 복귀 시 서비스 자동 제거
          └── 프로모션 비활성/없을 시 넛지 영역 미표시
```

| 화면 | 경로 | 사용자 | 핵심 기능 |
|------|------|--------|-----------|

| 메뉴 관리 | `/admin/menus` | 관리자 | 메뉴/카테고리 CRUD, 이미지 업로드 |

## 5. 실시간 통신 채널 요약

| 채널 | 방향 | 용도 |
|------|------|------|
| SSE (주문) | 서버 → 관리자 | 신규 주문 알림, 주문 목록 실시간 갱신 |
| SSE (상태) | 서버 → 고객 | 주문 상태 변경 실시간 반영 |
| SSE (직원호출) | 서버 → 관리자 | 직원 호출 알림 (테이블번호 + 사유) |
| SSE (장바구니) | 서버 → 고객 | 동행자 장바구니 변경 실시간 동기화 |
| SSE (세션) | 서버 → 고객 | 이용 완료 처리 시 세션 종료 안내 |

## 6. MVP 제외 범위

다음 기능은 현재 서비스 플로우에 포함되지 않음:

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 주방 전달 및 재고 관리
- 푸시 알림 / SMS / 이메일 알림
- 매출 리포트 및 데이터 분석 대시보드
- 예약, 리뷰, 다국어 지원
- 외부 플랫폼 연동 (배달, POS, 소셜 미디어)
