# User Stories - 테이블오더 서비스

> **형식**: 체크리스트 수용 기준 | **세분화**: 기능당 2~4개 | **우선순위**: 고객 플로우 최우선
> **Priority**: P0 (Must Have), P1 (Should Have), P2 (Nice to Have)

---

## Epic 1: 테이블 접속 (FR-C01) - 고객

### US-1.1: QR 코드로 테이블 접속 [P0]
**As a** 고객,
**I want to** QR 코드를 스캔하여 비밀번호 없이 즉시 메뉴 화면에 접속하고 싶다,
**So that** 대기 없이 바로 주문을 시작할 수 있다.

**Acceptance Criteria:**
- [ ] QR 코드 스캔 시 매장ID + 테이블번호가 포함된 URL로 이동
- [ ] 별도 로그인/비밀번호 없이 즉시 메뉴 화면 표시
- [ ] 매장명, 테이블 번호가 화면 상단에 표시
- [ ] 잘못된 QR(유효하지 않은 매장/테이블) 스캔 시 에러 안내

### US-1.2: 다중 사용자 동시 접속 [P0]
**As a** 고객,
**I want to** 동행자와 같은 테이블에서 각자의 기기로 동시 접속하고 싶다,
**So that** 각자 편하게 메뉴를 고르고 함께 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 같은 테이블 QR을 여러 기기에서 스캔 시 모두 정상 접속
- [ ] "현재 N명이 함께 주문 중" 안내 문구 실시간 표시
- [ ] 접속자 수 증가/감소 시 안내 문구 실시간 업데이트
- [ ] 테이블 단위 공유 장바구니 (모든 접속자가 전체 내용 확인 가능)

### US-1.3: 태블릿 자동 로그인 [P1]
**As a** 관리자,
**I want to** 테이블 태블릿에 매장/테이블 정보를 1회 설정하면 자동 로그인되게 하고 싶다,
**So that** 태블릿을 고객 전용 주문 기기로 상시 운영할 수 있다.

**Acceptance Criteria:**
- [ ] 초기 설정: 매장 식별자 + 테이블 번호 입력
- [ ] 설정 정보 로컬 저장 후 자동 로그인
- [ ] 브라우저 새로고침 시에도 자동 로그인 유지
- [ ] 16시간 세션 유지

## Epic 2: 메뉴 조회 (FR-C02) - 고객

### US-2.1: 카테고리별 메뉴 탐색 [P0]
**As a** 고객,
**I want to** 카테고리별로 메뉴를 탐색하고 싶다,
**So that** 원하는 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- [ ] 접속 시 메뉴 화면이 기본(홈)으로 표시
- [ ] 상단에 카테고리 탭 고정 (Sticky), 수평 스크롤 가능
- [ ] 카테고리 탭 클릭 시 해당 섹션으로 스크롤 이동
- [ ] 스크롤 시 현재 보이는 카테고리 탭 자동 하이라이트
- [ ] 카테고리 드롭다운으로 전체 목록 바텀시트 표시

### US-2.2: 메뉴 상세 정보 확인 [P0]
**As a** 고객,
**I want to** 메뉴의 이름, 가격, 설명, 이미지를 확인하고 싶다,
**So that** 주문 전에 충분한 정보를 얻을 수 있다.

**Acceptance Criteria:**
- [ ] 리스트형 레이아웃: 좌측 텍스트(뱃지+이름+가격+설명) + 우측 썸네일(100x100)
- [ ] 메뉴명(SemiBold), 가격(Bold), 설명(최대 2줄 말줄임)
- [ ] 이미지 없는 메뉴는 텍스트만 표시
- [ ] 인기/신규 뱃지 표시
- [ ] 메뉴 아이템 탭 시 상세 화면 이동

## Epic 15: 직원 호출 (FR-C09) - 고객/관리자

### US-15.1: 직원 호출 요청 (고객) [P0]
**As a** 고객,
**I want to** 직원을 호출하면서 무엇이 필요한지 미리 알려주고 싶다,
**So that** 직원이 필요한 것을 가져와서 빠르게 도움받을 수 있다.

**Acceptance Criteria:**
- [ ] 화면 우측 하단 플로팅 버튼으로 "직원호출" 상시 표시 (장바구니 바 위)
- [ ] 클릭 시 바텀시트: "무엇을 도와드릴까요?"
- [ ] 사유 선택지: 물/물티슈 요청, 수저/접시 요청, 기타 요청
- [ ] "기타 요청" 선택 시 간단한 텍스트 입력 가능
- [ ] 호출 완료 시 "호출했어요" 토스트 표시
- [ ] 연속 호출 방지 (30초 쿨다운, 쿨다운 중 버튼 비활성)

### US-15.2: 직원 호출 알림 수신 (관리자) [P0]
**As a** 관리자,
**I want to** 고객이 직원을 호출하면 어떤 테이블에서 무엇을 요청하는지 즉시 알고 싶다,
**So that** 필요한 것을 준비해서 바로 응대할 수 있다.

**Acceptance Criteria:**
- [ ] 직원호출 시 해당 테이블 카드에 호출 알림 표시 (사유 포함)
- [ ] SSE로 실시간 호출 알림 수신
- [ ] 호출 알림 시각적 강조 (색상/아이콘/애니메이션)
- [ ] "확인" 버튼 클릭 시 호출 알림 해제
- [ ] 여러 테이블에서 동시 호출 시 각각 독립 표시

## Epic 13: 메뉴 추천 (FR-C07) - 고객

### US-13.1: 함께 주문 추천 (페어링) [P1]
**As a** 고객,
**I want to** 현재 보고 있는 메뉴와 함께 많이 주문되는 메뉴를 알고 싶다,
**So that** 아무 생각 없이 "이거 좋은 조합인가보다!" 하고 바로 담을 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 상세 화면에 "손님들의 선택" 섹션 표시
- [ ] 해당 매장의 주문 데이터 기반 연관 메뉴 최대 3개 추천
- [ ] 각 추천 메뉴에 실제 함께 주문된 횟수 표시 (예: "326번 함께 주문됐어요")
- [ ] 추천 메뉴 클릭 시 해당 메뉴 상세로 이동
- [ ] 데이터 부족 시 (주문 이력 적을 때) 섹션 미표시

### US-13.2: 내 취향 맞춤 추천 (개인화) [P1]
**As a** 로그인 고객,
**I want to** 내 과거 주문 이력 기반으로 메뉴를 추천받고 싶다,
**So that** 새로운 매장에서도 내 취향에 맞는 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 화면 상단에 "{닉네임}님 맞춤 추천" 섹션 표시
- [ ] 이 매장 재방문 시: "지난번에 드신 메뉴" 표시 (재주문 유도)
- [ ] 유사 매장 주문 이력 기반: "이런 메뉴 어때요?" 추천
- [ ] 첫 방문(이력 없음) 시: 매장 인기 메뉴로 대체
- [ ] 추천 메뉴 클릭 시 해당 메뉴 상세로 이동
- [ ] 카카오 로그인 사용자 전용 (비로그인 시 미표시 — 현재 로그인 필수이므로 항상 표시)

## Epic 3: 장바구니 관리 (FR-C03) - 고객

### US-3.1: 공유 장바구니에 메뉴 담기 [P0]
**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 담고, 동행자가 담은 메뉴도 함께 확인하고 싶다,
**So that** 테이블 전체 주문 내용을 파악하고 중복 없이 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 상세 화면에서 수량 선택 후 장바구니 담기
- [ ] 장바구니에 메뉴 추가 시 토스트 알림 표시
- [ ] 하단 플로팅 바에 총 금액(전체 합산) + "장바구니 보기" 버튼 표시
- [ ] 같은 메뉴를 여러 명이 담으면 하나의 행에 병합, 아바타 아이콘 모두 표시
- [ ] 각 아이콘에 해당 사용자의 수량 표시, 총 수량은 전체 합산
- [ ] 한 명만 담은 메뉴는 아이콘 1개만 표시
- [ ] 아바타: 카카오 프로필 이미지 + 닉네임 / 본인은 "나" 라벨
- [ ] 누구나 모든 항목의 수량 변경/삭제 가능
- [ ] 총 금액 실시간 계산 (전체 접속자 메뉴 합산)
- [ ] 멤버가 메뉴를 담거나 삭제하면 실시간 반영

### US-3.2: 장바구니 실시간 동기화 [P0]
**As a** 고객,
**I want to** 동행자가 장바구니를 변경하면 내 화면에도 즉시 반영되길 원한다,
**So that** 항상 최신 주문 상태를 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 장바구니 데이터 서버 측 저장 (테이블 단위 공유)
- [ ] 멤버가 메뉴 추가 시 토스트: "[아이콘] 멤버가 {메뉴명}을 담았어요"
- [ ] 멤버가 메뉴 삭제 시 토스트: "[아이콘] 멤버가 {메뉴명}을 삭제했어요"
- [ ] 브라우저 새로고침 시 서버에서 장바구니 복원

## Epic 4: 주문 생성 (FR-C04) - 고객

### US-4.1: 주문 확정 [P0]
**As a** 고객,
**I want to** 공유 장바구니의 전체 메뉴를 최종 확인하고 주문을 확정하고 싶다,
**So that** 테이블 전체가 원하는 메뉴를 한 번에 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 장바구니 화면에서 전체 주문 내역 확인 (시간순 통합 리스트, 주문자 아이콘, 수량, 단가, 총 금액)
- [ ] "주문하기" 버튼 클릭 시 공유 장바구니 전체를 하나의 주문으로 서버 전송
- [ ] 주문 정보 포함: 매장ID, 테이블ID, 메뉴 목록, 총 금액, 세션ID
- [ ] 주문 성공 시: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트
- [ ] 주문 성공 시: 공유 장바구니 전체 비우기 (모든 접속자 화면 반영)

### US-4.2: 주문 실패 처리 [P0]
**As a** 고객,
**I want to** 주문 실패 시 원인을 알고 재시도할 수 있길 원한다,
**So that** 주문 내용을 잃지 않고 문제를 해결할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 실패 시 에러 메시지 표시
- [ ] 주문 실패 시 장바구니 내용 유지
- [ ] 네트워크 오류 시 재시도 안내
- [ ] 세션 만료 시 재접속 안내

## Epic 5: 주문 내역 조회 (FR-C05) - 고객

### US-5.1: 현재 세션 주문 내역 조회 [P0]
**As a** 고객,
**I want to** 현재 테이블의 전체 주문 내역을 확인하고 싶다,
**So that** 무엇을 주문했는지, 주문 상태가 어떤지 알 수 있다.

**Acceptance Criteria:**
- [ ] 상단 "주문내역" 버튼으로 접근
- [ ] 현재 테이블 세션의 전체 주문 표시 (모든 접속자 주문 포함)
- [ ] 주문별: 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태
- [ ] 주문 상태: 대기중 / 준비중 / 완료
- [ ] 주문 시간 순 정렬
- [ ] 이용 완료 처리된 이전 세션 주문은 제외

### US-5.2: 주문 상태 실시간 업데이트 [P1]
**As a** 고객,
**I want to** 주문 상태 변경을 실시간으로 확인하고 싶다,
**So that** 주문이 어떻게 진행되고 있는지 바로 알 수 있다.

**Acceptance Criteria:**
- [ ] 관리자가 주문 상태를 변경하면 고객 화면에 실시간 반영
- [ ] 상태 변경 시 시각적 표시 (색상/아이콘 변경)

## Epic 6: 매장 인증 (FR-A01) - 관리자

### US-6.1: 관리자 로그인 [P0]
**As a** 관리자,
**I want to** 매장 식별자와 계정 정보로 로그인하고 싶다,
**So that** 내 매장의 관리 시스템에 안전하게 접근할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자 + 사용자명 + 비밀번호 입력
- [ ] 로그인 성공 시 JWT 토큰 발급, 주문 대시보드로 이동
- [ ] 16시간 세션 유지, 브라우저 새로고침 시 세션 유지
- [ ] 16시간 후 자동 로그아웃
- [ ] 로그인 실패 시 에러 메시지 표시
- [ ] 로그인 시도 제한 (brute-force 방어)

## Epic 7: 실시간 주문 모니터링 (FR-A02) - 관리자

### US-7.1: 테이블별 주문 대시보드 [P0]
**As a** 관리자,
**I want to** 모든 테이블의 주문 현황을 그리드 형태로 한눈에 보고 싶다,
**So that** 매장 전체 상황을 즉시 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블별 카드 형태 그리드 레이아웃
- [ ] 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- [ ] 신규 주문 시 해당 카드 시각적 강조 (색상/애니메이션)
- [ ] SSE 기반 실시간 업데이트 (2초 이내)
- [ ] 테이블별 필터링 기능

### US-7.2: 주문 상태 관리 [P0]
**As a** 관리자,
**I want to** 주문 상태를 변경하고 상세 내역을 확인하고 싶다,
**So that** 주문 처리 진행 상황을 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블 카드 클릭 시 해당 테이블 전체 주문 목록 상세 보기
- [ ] 주문별: 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태
- [ ] 주문 접수 시 자동으로 "대기중" 상태 설정
- [ ] 관리자가 수동으로 주문 건 단위 상태 변경: 대기중 → 준비중 → 완료
- [ ] 10분 이상 "대기중" 상태인 주문에 강조 알림 표시 (색상/깜박임)
- [ ] 상태 변경 시 고객 화면에도 실시간 반영

### US-7.3: 신규 주문 실시간 알림 [P0]
**As a** 관리자,
**I want to** 새 주문이 들어오면 즉시 알림을 받고 싶다,
**So that** 주문을 놓치지 않고 빠르게 처리할 수 있다.

**Acceptance Criteria:**

## Epic 8: 테이블 관리 (FR-A03) - 관리자

**Acceptance Criteria:**
- [ ] 주문 삭제 버튼 클릭 시 확인 팝업 표시
- [ ] 확인 후 주문 즉시 삭제
- [ ] 삭제 후 테이블 총 주문액 자동 재계산
- [ ] 성공/실패 피드백 표시

### US-8.3: 테이블 이용 완료 처리 [P0]

**Acceptance Criteria:**
- [ ] "이용 완료" 버튼 클릭 시 확인 팝업 표시
- [ ] 확인 후 해당 세션 주문 내역 → 과거 이력으로 이동
- [ ] 테이블 현재 주문 목록 및 총 주문액 0으로 리셋
- [ ] 새 고객 접속 시 이전 주문 없이 시작
- [ ] 성공/실패 피드백 표시

### US-8.4: 과거 주문 내역 조회 [P1]
**As a** 관리자,
**I want to** 테이블의 과거 주문 이력을 확인하고 싶다,
**So that** 이전 매출/주문 현황을 파악할 수 있다.

**Acceptance Criteria:**
- [ ] "과거 내역" 버튼으로 접근
- [ ] 테이블별 과거 주문 목록 표시 (시간 역순)
- [ ] 각 주문: 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각
- [ ] 날짜 필터링 기능
- [ ] "닫기" 버튼으로 대시보드 복귀

## Epic 14: 서비스 프로모션 (FR-A07 + FR-C08) - 관리자/고객

### US-14.1: 서비스 프로모션 설정 (관리자) [P1]
**As a** 관리자,
**I want to** "N원 이상 주문 시 OOO 서비스" 프로모션을 설정하고 싶다,
**So that** 추가 주문을 자연스럽게 유도하고 고객 만족도를 높일 수 있다.

**Acceptance Criteria:**
- [ ] 프로모션 등록: 목표 금액 + 서비스 메뉴/혜택 이름 입력
- [ ] 동시에 여러 프로모션 등록 가능 (예: 2만원→음료, 3만원→디저트)
- [ ] 프로모션 활성/비활성 토글
- [ ] 프로모션 수정/삭제 가능
- [ ] 필수 필드 검증 (목표 금액 > 0, 서비스 내용 필수)

### US-14.2: 서비스 프로모션 넛지 표시 (고객) [P1]
**As a** 고객,
**I want to** 서비스 혜택까지 얼마나 남았는지 자연스럽게 알고 싶다,
**So that** "조금만 더 시키면 서비스 받겠네!" 하고 추가 주문하게 된다.

**Acceptance Criteria:**
- [ ] 장바구니 플로팅 바 영역에 프로그레스 바 + 남은 금액 표시
- [ ] 예시: "3,000원만 더 주문하면 마카롱 서비스!" + 프로그레스 바
- [ ] 장바구니 금액 변동 시 실시간 업데이트 (WebSocket 연동)
- [ ] 여러 프로모션이 있으면 가장 가까운 목표부터 표시
- [ ] 프로모션 없을 시 넛지 영역 미표시

### US-14.3: 서비스 달성 축하 (고객) [P1]
**As a** 고객,
**I want to** 서비스 혜택 금액을 달성하면 기분 좋은 알림을 받고 싶다,
**So that** 보상받는 느낌으로 만족감을 얻을 수 있다.

**Acceptance Criteria:**
- [ ] 목표 금액 달성 시 축하 애니메이션 + 문구 (예: "마카롱도 드려요! 🎉")
- [ ] 서비스 메뉴가 장바구니에 자동 추가 (0원 표시, "서비스" 태그)
- [ ] 장바구니 금액이 다시 목표 미만으로 내려가면 서비스 자동 제거 + 안내
- [ ] 다음 단계 프로모션이 있으면 자연스럽게 다음 목표 표시

**Acceptance Criteria:**
- [ ] 로그인 고객의 주문 시 고객 프로필과 주문 자동 연결
- [ ] 다른 매장 방문 시에도 같은 계정에 이력 누적
- [ ] 선호 메뉴 자동 분석 (주문 빈도 기반)
- [ ] 방문 매장, 방문 횟수, 총 주문액 자동 집계
