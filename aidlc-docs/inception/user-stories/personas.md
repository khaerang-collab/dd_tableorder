# User Personas - 테이블오더 서비스

---

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **접속 방식** | QR 코드 스캔 (개인 스마트폰) 또는 테이블 태블릿 |
| **목표** | 빠르고 편리하게 메뉴를 주문하고, 주문 상태를 확인 |
| **기술 수준** | 일반 (스마트폰 기본 사용 가능) |
| **핵심 니즈** | 대기 없이 즉시 주문, 직관적 UI, 동행자와 함께 주문 |
| **불편 포인트** | 복잡한 인증 과정, 느린 페이지 로딩, 주문 상태 불확실 |
| **사용 빈도** | 매장 방문 시마다 (비정기) |
| **기기** | 스마트폰 (모바일 브라우저) |

### 고객 시나리오
1. 테이블에 앉아 QR 코드를 스캔하여 즉시 메뉴 화면 접속
2. 카테고리별 메뉴를 탐색하고 장바구니에 담기
3. 동행자도 같은 QR을 스캔하여 각자 주문
4. 주문 확정 후 주문 상태 확인
5. 추가 주문 시 동일 과정 반복

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (대표 관리자) |
| **역할** | 매장 운영자 / 점주 |
| **접속 방식** | 관리자 로그인 (PC 또는 태블릿 브라우저) |
| **목표** | 실시간 주문 모니터링, 메뉴 관리, 테이블 운영 효율화 |
| **기술 수준** | 중급 (기본 웹 서비스 사용 가능) |
| **핵심 니즈** | 주문 즉시 확인, 테이블별 상황 파악, 메뉴 빠른 수정 |
| **불편 포인트** | 주문 누락, 테이블 혼선, 복잡한 관리 도구 |
| **사용 빈도** | 매일 (영업 시간 내 상시) |
| **기기** | PC / 태블릿 (브라우저) |

### 관리자 시나리오
1. 매장 오픈 시 관리자 로그인 (16시간 세션)
2. 대시보드에서 테이블별 주문 현황 실시간 모니터링
3. 신규 주문 접수 시 상태 변경 (대기중 → 준비중 → 완료)
4. 필요 시 주문 삭제 또는 테이블 이용 완료 처리
5. 메뉴 추가/수정/삭제 및 카테고리 관리

## 페르소나-스토리 매핑

| 페르소나 | 관련 Epic |
|----------|-----------|
| 고객 (김민수) | Epic 1~5 (접속, 메뉴 조회, 장바구니, 주문, 주문 내역) |
| 관리자 (박서연) | Epic 6~10 (인증, 모니터링, 테이블 관리, 메뉴 관리, 이미지) |
