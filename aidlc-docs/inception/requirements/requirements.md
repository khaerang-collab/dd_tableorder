## 3. 기능적 요구사항

### 3.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 접속 및 세션 관리
- **다중 사용자 동시 접속 지원**: 한 테이블에 여러 명이 각자의 기기로 동시 접속 가능
- **접속 방식 1 - 태블릿 자동 로그인**:
  - 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호)
  - 로그인 정보 로컬 저장 후 자동 로그인
- **접속 방식 2 - QR 코드 접속**:
  - 각 테이블에 고유 QR 코드 비치 (URL에 매장ID + 테이블번호 포함)
  - 고객이 QR 스캔 시 비밀번호 없이 즉시 메뉴 화면 접속
  - 별도 앱 설치 없이 브라우저에서 바로 이용
- **동시 접속 안내**:
  - 같은 테이블에 여러 명이 접속 중일 때 "현재 N명이 함께 주문 중" 안내 문구 표시
  - 접속자 수 실시간 업데이트
- **세션 관리**:
  - 테이블 세션: 테이블 단위로 관리 (첫 주문 시 시작 ~ 이용 완료 시 종료)
  - 개인 세션: 기기별 독립 세션 (장바구니는 테이블 단위 공유, 담은 사람 구분)
  - 16시간 세션 유지

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동 (상단 고정 탭 + 드롭다운)
- **리스트형 레이아웃** (텍스트 좌측 + 썸네일 우측 배치)
- 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- **테이블 공유 장바구니**: 같은 테이블의 모든 접속자가 장바구니를 공유
  - 각자가 담은 메뉴를 "내 메뉴" / "멤버 메뉴"로 구분 표시
  - 주문 확정 전까지 모든 접속자가 전체 장바구니 내용 확인 가능
  - 자신이 담은 메뉴만 수정/삭제 가능
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산 (전체 합산)
- 장바구니 비우기 (자신의 메뉴만)
- 서버 측 장바구니 저장 (공유를 위해 서버 동기화)
- 주문 확정 시 장바구니 전체를 하나의 주문으로 전송

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 → 5초 후 장바구니 비우기 및 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 식별, 테이블 식별, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 **전체 주문** 표시 (같은 테이블의 모든 접속자 주문 포함)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외
- 페이지네이션 또는 무한 스크롤

### 3.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호 설정, QR 코드 생성, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 피드백
- **테이블 세션 관리**: 
  - 세션 시작: 해당 세션의 첫 주문 시 자동 시작
  - 세션 종료(이용 완료): 주문 내역 → 과거 이력 이동, 현재 목록/총액 리셋
  - 새 고객은 이전 주문 없이 시작
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록(시간 역순), 날짜 필터링

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (조회/등록/수정/삭제)
- 메뉴 항목: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증
- **카테고리 CRUD**: 카테고리 추가/수정/삭제 가능 (동적 관리)

#### FR-A05: 이미지 업로드
- 서버에 이미지 파일 업로드 및 로컬 저장
- 메뉴 등록/수정 시 이미지 첨부 가능

## 4. 비기능적 요구사항

### NFR-01: 성능
- SSE 기반 실시간 주문 알림: 2초 이내 표시
- 페이지 로딩: 3초 이내
- API 응답 시간: 500ms 이내

### NFR-02: 확장성
- 초기 MVP: 다수 매장 지원 (10개 이상)
- 멀티테넌트 아키텍처 (매장별 데이터 격리)

### NFR-03: 보안 (Security Baseline 적용)
- SECURITY-01 ~ SECURITY-15 전체 규칙 적용
- JWT 기반 인증/인가
- 비밀번호 bcrypt 해싱
- 입력값 검증 (모든 API 파라미터)
- CORS 설정, HTTP 보안 헤더
- Rate limiting
- 로그인 시도 제한 (brute-force 방어)

### NFR-04: 가용성
- 서비스 가동 시간 목표: 99.5%

### NFR-05: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 기반 레이아웃
- 명확한 시각적 계층 구조

## 5. 제외 범위

다음 기능은 MVP에서 제외:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| **테이블 세션** | 고객이 테이블에 앉아 첫 주문 시작 ~ 이용 완료 처리까지의 기간. 테이블 단위로 하나만 존재 |
| **개인 세션** | 한 테이블에 접속한 개별 기기/사용자의 세션. 장바구니는 테이블 단위 공유하되 담은 사람을 구분 |
| **이용 완료** | 관리자가 해당 테이블 세션을 종료하는 행위 |
| **QR 코드 접속** | 테이블에 비치된 QR 코드를 스캔하여 개인 기기로 주문 시스템에 접속하는 방식 |
| **SSE** | Server-Sent Events, 서버→클라이언트 단방향 실시간 통신 |
| **MVP** | Minimum Viable Product, 최소 기능 제품 |

### 8.4 상단 네비게이션 바

```
+--------------------------------------------------+
| [직원호출 버튼]                      [주문내역] |

```

- **좌측**: 직원호출 버튼 (벨 아이콘 + "직원호출" 텍스트)
- **우측**: "주문내역" 텍스트 버튼
- 폰트: T5(17px), SemiBold(600)
- 색상: Cool Gray 700(`#5F6972`) / Cool Gray 900(`#2C3744`)

### 8.5 매장 정보 헤더

```
+--------------------------------------------------+
| 매장명 (T3, Bold)              테이블: N번 테이블 |
| 멤버도 QR 찍고 [함께 주문]해요    (T7, Grey600)  |
+--------------------------------------------------+
| [공지 아이콘] 매장 공지 메시지 (T6)               |
+--------------------------------------------------+
```

- **매장명**: T3(22px), Bold(700), Cool Gray 900(`#2C3744`)
- **함께 주문 안내**: T5(17px), Medium(500), Cool Gray 700(`#5F6972`)
  - "함께 주문" 텍스트 하이라이트 처리 (Yellow 400 배경 또는 Bold 강조)
  - 동시 접속 시: "현재 N명이 함께 주문 중"
- **테이블 번호**: 우측 상단, T7(13px), Cool Gray 500(`#80878F`)
- **매장 공지**: 메가폰 이모지 + 텍스트, T6(15px), Cool Gray 700(`#5F6972`)

### 8.6 카테고리 탭 (Sticky)

```
+--------------------------------------------------+
| [카테고리1] [카테고리2] [*카테고리3*] [카테고리4] |
|                         ─── (인디케이터)          |
+--------------------------------------------------+
```

- **상단 고정** (position: sticky/fixed, z-index: 10)
- **수평 스크롤** 가능한 탭 리스트
- **선택된 탭**: Bold(700), Cool Gray 900(`#2C3744`), 하단 인디케이터 라인 (Yellow 400)
- **비선택 탭**: SemiBold(600), Cool Gray 500(`#80878F`)
- **폰트**: T5(17px)
- **패딩**: 아이템 간 12px, 타이틀 상하 12px/14px
- **드롭다운 버튼**: 카테고리 전체 목록 바텀시트 표시

### 8.7 메뉴 아이템 (리스트형)

```
+--------------------------------------------------+
| [인기] 뱃지                                      |
| 메뉴명 (T5, SemiBold)         +----------------+ |
| 가격 (T6, Bold)               |  썸네일 이미지  | |
| 메뉴 설명 (T7, 최대 2줄)      |  100x100px     | |
|                                | border-r: 18px | |
|                                +----------------+ |
+--------------------------------------------------+
| ─── hairline 구분선 ───                           |
+--------------------------------------------------+
```

- **레이아웃**: 수평 배치 (텍스트 좌측, 이미지 우측)
- **이미지**: 100x100px, border-radius 18px, object-fit: cover
- **이미지 없는 메뉴**: 텍스트만 표시 (이미지 영역 생략)
- **뱃지**: 메뉴명 위에 표시
  - 인기: Blue 300(`#0077D1`) 배경, 흰색 텍스트, Bold
  - 신규: Red 300(`#F55251`) 배경, 흰색 텍스트, Bold
  - 둥근 pill 형태 (border-radius)
- **메뉴명**: T5(17px), SemiBold(600), Cool Gray 900(`#2C3744`)
- **가격**: T6(15px), Bold(700), Cool Gray 900(`#2C3744`)
- **설명**: T7(13px), Regular(400), Cool Gray 500(`#80878F`), 최대 2줄 (말줄임 처리)
- **패딩**: 상하 8px, 좌우 24px
- **아이템 간 구분**: 1px hairline border (Cool Gray 200 `#E5E8EB`)
- **카테고리 간 구분**: 16px 높이 Cool Gray 100(`#EDF0F3`) 배경 구분선

### 8.9 하단 장바구니 바 (Floating)

```
+--------------------------------------------------+
|     [ 🛒 00,000원 장바구니 보기 ]                 |
+--------------------------------------------------+
```

- **하단 고정** 플로팅 바
- 장바구니에 아이템이 있을 때만 표시
- 총 금액 표시 + "장바구니 보기" 텍스트
- Yellow 400(`#FFED49`) CTA 버튼, Cool Gray 900(`#2C3744`) 텍스트
- Safe Area 하단 여백 적용

### 8.10 하단 매장 정보 (Footer)

- "가게정보·원산지" 제목: T7(13px), SemiBold(600), Cool Gray 700(`#5F6972`)
- 상호명, 가게주소, 원산지: T7(13px), Regular(400), Cool Gray 500(`#80878F`)
- 메뉴 목록 최하단에 위치

### 8.11 인터랙션 패턴

| 인터랙션 | 동작 |
|----------|------|
| **메뉴 아이템 탭** | 메뉴 상세 화면으로 이동 (옵션 선택 등) |
| **장바구니 바 탭** | 장바구니 페이지로 이동 |
| **카테고리 탭 탭** | 해당 카테고리 섹션으로 스크롤 이동 |
| **카테고리 드롭다운** | 전체 카테고리 바텀시트 표시 |
| **직원호출 탭** | "무엇을 도와드릴까요?" 바텀시트 표시 |
| **스크롤** | 카테고리 탭 자동 하이라이트 (현재 보이는 섹션 기준) |
| **멤버 장바구니 변경** | 토스트 알림: "멤버가 {메뉴} 메뉴를 담았어요" |

### 8.12 토스트 알림

- 화면 상단 또는 하단에 일시적으로 표시
- 반투명 다크 배경: Dimmed 40% `rgba(0, 0, 0, 0.4)`
- 자동 소멸 (약 3초)
- 사용 예:
  - "멤버가 {메뉴} 메뉴를 담았어요"
  - "장바구니에 메뉴를 추가했어요"
  - "메뉴를 삭제했어요"
  - "품절된 상품이에요"

### 8.13 주문 완료 화면

- "총 N개 주문 완료!" 메시지
- 5초 후 "메뉴판으로" 자동 리다이렉트
- 하단에 "메뉴판으로" 버튼

### 8.14 주문 내역 화면

- "주문 내역" 타이틀
- 총 N개 카운트 표시
- 주문별 상세 정보 리스트
- 하단에 "추가 주문하기" / "메뉴 보기" 버튼
